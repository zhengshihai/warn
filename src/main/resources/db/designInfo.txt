#在晚归记录表（late_return）中
    -status 字段：
    这是记录的整体状态
    PENDING（待处理）：表示这条晚归记录刚刚被记录，还没有被宿管或管理员处理
    PROCESSED（已处理）：表示这条晚归记录已经被相关人员查看并处理完成
    主要用于快速筛选哪些记录需要处理，哪些已经处理过

    -process_status 字段：
    Apply to designInfo.t...
    这是具体的处理结果状态
    PENDING（待处理）：还未做出处理决定
    APPROVED（已通过）：晚归理由被认可，比如参加学校活动、图书馆学习等正当理由
    REJECTED（已驳回）：晚归理由不被接受，可能需要进一步的教育或处理
    用于记录对晚归原因的审核结果

   - process_remark 字段：
    Apply to designInfo.t...
    用于记录处理人员的具体处理意见
    可以记录为什么通过或驳回
    可以记录对学生的具体要求或建议
    便于后续跟踪和查证

    -实际使用场景示例：
    学生晚归后：
    status = 'PENDING'
    process_status = 'PENDING'
    process_remark = NULL
    宿管审核通过正当理由的情况：
    status = 'PROCESSED'
    process_status = 'APPROVED'
    process_remark = '学生因参加学校组织的文艺汇演晚归，已核实属实'
    宿管驳回无正当理由的情况：
    status = 'PROCESSED'
    process_status = 'REJECTED'
    process_remark = '无正当理由，建议加强时间管理意识，避免再次晚归'

#system_rule和 warning_rule 这两张表的作用

-system_rule（系统规则表）：
    这是一个系统级的配置表，用于存储系统运行所需的各种参数配置，比如：
    晚归时间点设置：LATE_RETURN_TIME = 23:00:00
    连续晚归阈值：MAX_CONTINUOUS_LATE = 3
    各类通知模板：

    特点：
        采用key-value形式存储
        可以随时修改系统参数而不需要重启
        适合存储全局性的配置项

-warning_rule（预警规则表）：
    这是一个业务级的规则表，用于配置具体的预警策略，比如：
    一周三次预警：
    连续晚归预警：

    特点：
        每条记录代表一个完整的预警规则
        可以设置启用/禁用状态
        支持多种预警条件组合

-两者的主要区别：
    使用场景：
    system_rule：系统运行参数配置，如时间点、模板等
    warning_rule：业务预警规则配置，如预警条件、通知方式等
    数据结构：
    system_rule：简单的key-value结构
    warning_rule：复杂的规则结构，包含多个业务字段
    使用方式：
    system_rule：系统启动时加载，运行时可修改
    warning_rule：预警任务执行时动态查询使用
    配置粒度：
    system_rule：系统级别，全局生效
    warning_rule：规则级别，可以针对不同情况配置不同规则

#查看界面原型
    使用IDEA打包项目：
    右键点击项目
    选择"Build Module 'warn'"或使用Maven面板执行mvn clean package
    打包后的war文件将在target目录下生成，名为warn-1.0-SNAPSHOT.war
    部署到本地Tomcat：
    将生成的war包复制到Tomcat的webapps目录下
    或者直接在IDEA中配置Tomcat运行：
    点击"Edit Configurations"
    添加"Tomcat Server" -> "Local"
    在"Deployment"标签页添加war包
    启动Tomcat后，您可以通过以下URL访问：
    登录页面：http://localhost:8080/warn/login
    仪表盘：http://localhost:8080/warn/dashboard
    晚归记录列表：http://localhost:8080/warn/late-records
    预警详情：http://localhost:8080/warn/warning-detail
    学生自助查询：http://localhost:8080/warn/student-portal
    需要注意的几点：
    确保本地MySQL服务已启动，并已执行了schema.sql和data.sql脚本
    确保Tomcat版本与项目Java版本兼容（建议使用Tomcat 8.5+）
    第一次访问时使用以下测试账号：
    管理员：username=admin, password=admin
    宿管：username=dormitory1, password=admin
